Game of Life

Introduction

This is my implementation of Conway's Game of Life, a cellular automation game where the outcome is determined by the initial seed. Cells can produce a new cell, remain alive, or die based on simple rules:

  • Any cell with fewer than 2 neighbors dies, as if by underpopulation.
  • Any live cell with 2 or 3 live neighbors lives to the next generation.
  • Any live cell with more than 3 neighbors dies, as if by overpopulation.
  • Any dead cell with exactly 3 live neighbors becomes a live cell, if by reproduction.

Implentation

Technologies Used: Java

In designing this project, I ensured that all variables affecting the game were separated out and immutable. This allows for easy adjustments to observe how the game plays out, while ensuring that the inital parameters remain the same.

I chose an object-oriented approach, creating cell objects that could be active or inactive. During the initial run, the program refers to the variables to determine where active cells can spawn, their size, and the chance that a cell in the spawn box is active. Based on the size of the cells, I generate a list of possible x and y coordinates and iterate through these lists to create the cell objects and generate the first cell generation.

To apply the rules to the current generation, I created methods to check if a cell's neighbors were active. To handle potential edge cases, I created attributes for the cell's number of active neighbors and whether the cell should live or die in the next generation. The purpose of the "flip active" attribute is to ensure that the status of the current generation is not changed until the entire screen is analyzed.

After enumerating the current generation of cells, I create the next generation by iterating through the cells. If the "flip active" variable is true, the cell's state is flipped. This process repeats indefinitely, completing the Game of Life.
